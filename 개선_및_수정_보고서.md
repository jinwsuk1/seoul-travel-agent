# 🍜 서울 맛집 AI 여행 에이전트 개선 및 수정 보고서

본 보고서는 서울 맛집 AI 추천 서비스의 시각적 완성도(UI/UX) 향상 및 검색 정확도(백엔드 로직) 개선을 위해 진행한 모든 작업 내역을 상세히 정리한 문서입니다.

---

## 1. 프론트엔드 UI/UX 디자인 개선 (app.py)

사용자가 서비스 접속 시 첫눈에 몰입할 수 있도록 프리미엄 다크 테마 및 애니메이션 디자인을 적용했습니다.

### ① 입력창 글래스모피즘(Glassmorphism) 고도화
- **적용 화면**: 웰컴 화면 중앙 검색창 및 하단 채팅 입력창
- **구현 특징**:
  - 반투명 슬레이트 배경(`rgba(15, 23, 42, 0.65)`)과 초정밀 블러 필터(`backdrop-filter: blur(16px)`) 적용.
  - 마우스 호버 및 포커스(Focus) 상태일 때 인디고(`indigo-400`) 계열의 부드러운 광채 효과(Soft Glow) 애니메이션이 일어나도록 보완하여 반응성을 높였습니다.

### ② 세로 적층형(Vertical Stacked) 프리미엄 카드 목록
- **적용 화면**: 맛집 추천 결과 출력 영역
- **구현 특징**:
  - 가로 슬라이더(캐러셀) 방식의 정보 유실 문제를 보완하기 위해, 한눈에 비교하기 편리한 세로 나열형 구조로 전면 전환했습니다.
  - 카드가 화면에 나타날 때 순차적(0.05초, 0.18초, 0.30초 간격)으로 부드럽게 솟아오르며 등장하는 **페이드인 업(Fade-in Up) 애니메이션**을 구현하여 고급스러운 감각을 부여했습니다.
  - 마우스 호버 시 이미지 줌인 효과(Scale 1.03) 및 GraphRAG 추천 대안 매장 하이라이트 인디고 패널, 커스텀 SVG 지도 연결 버튼을 제작했습니다.

## 2. 백엔드 검색 및 이미지 최적화 개선 (travel_agent.py)

맛집 검색 시 편의점이 추천되는 현상을 방지하고, 고품질 실물 사진을 보여주기 위한 정밀 필터 및 스코어링 알고리즘을 백엔드에 추가 적용했습니다.

### ① 편의점(CU, GS25 등) 검색 제외 필터 구축
- **문제 원인**: 공공 인허가 DB상 다수의 편의점이 `패스트푸드` 또는 `휴게음식점`으로 등록되어 있어 "패스트푸드 추천" 요청 시 편의점이 우선 노출되던 한계가 존재했습니다.
- **해결 조치**:
  - 사용자의 검색어에 직접적으로 "편의점", "CU", "GS25" 등 편의점 키워드가 명시적으로 들어있지 않은 모든 검색 요청에 한해, 결과 목록에서 편의점 브랜드 매장을 강제 배제시키는 차단 필터를 `retriever_agent`에 추가했습니다.

### ② 이미지 매칭 스코어링 알고리즘 도입
- **문제 원인**: 기존에는 실시간 이미지 검색 후보 10개 중 무조건 첫 번째 블로그 사진을 가져왔기 때문에 지도 캡처, 약도, 주차장, 메뉴판 등 저품질 이미지가 그대로 노출되는 문제가 있었습니다.
- **해결 조치**:
  - **부적절 키워드 벌점 (-30점)**: 제목/링크에 `지도, 약도, 주차장, 메뉴판, 영업시간` 등 텍스트 중심 캡처본이 감지되면 벌점을 부여하여 필터링합니다.
  - **대표 키워드 가점 (+10점)**: 제목에 `대표, 외관, 간판, 음식, 실내` 등의 단어가 들어있거나 실제 매장명이 정확히 일치하면 가점을 부여합니다.
  - **종횡비(Aspect Ratio) 가점 (+5점)**: 가로/세로 비율이 황금 비율(1.1 ~ 1.7)인 대칭형 사진에 가점을 주어 세로형 배너 광고 등을 차단합니다.
  - 최종적으로 누적 점수가 가장 높은 **최적의 대표 이미지 1장**만 선별 렌더링합니다.

## 3. 안정성 및 예외 처리 이력 (롤백 조치)

- **Qdrant DB 잠금(Lock) 해제**:
  - 백엔드 파일 실시간 리로드를 위해 추가했던 `importlib.reload` 구문이 Qdrant Local DB의 중복 접근 파일 잠금 에러(`RuntimeError`)를 유발하는 현상을 포착했습니다.
  - 이에 따라 해당 코드를 즉시 삭제하여 이전의 안전한 단독 로드 상태로 롤백을 마쳤으며, 데이터 오염 방지 및 정상 구동을 보장했습니다.
  - (※ 백엔드 로직 수정 반영 시에는 Streamlit 터미널에서 `Ctrl+C` 후 `streamlit run app.py` 재실행이 필요합니다.)

## 4. 파일별 수정 코드 요약

### [app.py] (수정 완료)
- 글래스모피즘 인풋 CSS 스타일 추가 및 전송 버튼 호버 전환.
- 단순 마크다운 응답에서 `parse_restaurants()` 연계 `render_restaurant_cards()` 세로 카드 HTML 렌더링 체계로 변경.

### [travel_agent.py] (수정 완료)
- `retriever_agent` 내 편의점 제외 조건 필터링 루프 구현.
- `naver_image_search` 내 이미지 메타데이터(제목, 비율) 분석 기반 실시간 스코어링 정렬 알고리즘 적용.

### [.streamlit/config.toml] (신규 추가)
- Streamlit 기본 다크 모드 속성 정의 (`base="dark"`, 배경색 `#0b0d14` 고정)하여 흰색 인풋 현상 원천 예방.
